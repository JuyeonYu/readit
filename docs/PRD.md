# PRD – 링크 기반 읽음 알림 서비스 (가칭: 읽었어?)

## 1. 개요

### 문제 정의

메신저나 이메일로 메시지를 보냈을 때 **상대가 읽었는지 여부를 알 수 없는 상황**이 잦다. 특히 링크, 공지, 긴 텍스트를 보낼 경우 상대의 확인 여부를 알기 어렵다.

### 해결책

내용을 **전용 링크로 공유**하고, 수신자가 링크를 열어 ‘읽기’를 명시적으로 선택하면 **송신자에게 익명 기반의 읽음 알림**을 제공한다.

### 핵심 가치 제안

* 설치 불필요 (웹 기반)
* 로그인 없이도 사용 가능
* 수신자에게 부담 없는 UX
* 개인정보 최소 수집

---

## 2. 타겟 사용자 (Persona)

### 1차 타겟
일상에서 중요한 메시지를 보냈지만, 상대가 확인했는지 알 수 없어 불안함을 느끼는 개인 사용자


### 2차 타겟
링크 기반으로 정보를 자주 공유하며, 전달 여부를 명확히 확인하고 싶은 파워 유저

---

## 3. 사용자 시나리오

### 송신자 플로우

1. 텍스트 입력
2. 링크 생성
3. 메신저/메일로 링크 공유
4. 누군가 읽으면 알림 수신

### 수신자 플로우

1. 링크 접속
2. “OOO님이 보낸 메시지입니다. 읽을까요?” 안내
3. [읽기] 버튼 선택
4. 콘텐츠 열람

---

## 4. 핵심 기능 (MVP)

### 4.1 콘텐츠 생성

* 텍스트 입력
* 선택 옵션

  * 1회 읽기
  * 만료 시간
  * 비밀번호 (선택)

### 4.2 읽기 UX

* 프리뷰 화면
* 읽기 버튼을 통한 명시적 액션
* 읽기 시 서버 이벤트 기록

### 4.3 읽음 알림

* 송신자에게 알림 제공

  * 이메일
  * 웹 알림 (MVP에서는 이메일 우선)
* 알림 정보

  * 읽은 시각
  * 총 읽은 횟수 (개인 식별 없이)

### 4.4 익명성 & 개인정보

* IP 저장 ❌
* User-Agent 최소 수집 (통계용)
* 쿠키 기반 중복 읽기 추정

---

## 5. 비기능 요구사항

### 보안
* 토큰 기반 접근
* 추측 불가능한 URL

### 성능

* 링크 열기 TTFB < 300ms
* 읽기 이벤트 처리 비동기화

### 확장성
* 웹 우선 → 이후 모바일/PWA 확장 가능

---

## 6. 수익 모델 (초기 가설)

### 무료
* 월 n개 링크 생성
* 기본 읽음 알림 (이메일)
* 만료 시간 기본값

### 유료 (후보)
#### 구독
* 고급 옵션 (만료, 비밀번호, 읽기 제한)
* 광고 제거
* 알림 채널 확장 (Slack, Webhook)

#### 일회성
링크 단위 과금
구독 없이 필요할 때만 결제
링크 n개 묶음 단위 구매
이벤트성 / 일회성 사용자 대응

### 실험적 모델
* 송신자 작성 버튼 누르면 30초 광고 노출

---

## 7. 지표 (Success Metrics)

* 링크 생성 → 읽기 전환율
* 읽기 버튼 클릭률
* 재방문율 (송신자 기준)
* 링크당 평균 읽기 횟수

---

## 8. 리스크 및 고려사항

* 수신자가 ‘감시당한다’는 인상을 받지 않도록 UX 주의
* 읽기 버튼 문구와 카피가 핵심
* 알림 과다 방지 필요

---

## 9. 향후 로드맵 (Post-MVP)

* 읽음 통계 대시보드
* 팀 공유 기능
* 링크 템플릿
* API / Webhook 제공

---

## 10. 결정 사항 요약

* 웹 기반, 로그인 최소화
* 익명성 우선
* 읽기 버튼 UX에 집중
* 빠른 실험과 지표 기반 개선
